#include <sys/stat.h>

<sys/stat.h> 是 Unix/Linux 系统编程中用于 文件元数据操作 的核心头文件，提供了 文件属性检查、权限管理、文件创建 等功能。

1. 核心功能

功能关键函数/宏用途
文件信息获取stat(), fstat(), lstat()获取文件元数据
文件权限检查S_ISDIR(), S_ISREG() 等宏判断文件类型
权限位操作S_IRUSR, S_IWGRP 等宏设置/检查文件权限
文件创建mkdir(), mkfifo()创建目录或命名管道
文件模式修改chmod(), fchmod()修改文件权限

2. 详细函数解析

2.1 文件信息获取

(1) stat() - 通过路径获取文件信息

  • 功能：获取文件元数据（如大小、权限、时间戳）。

  • 示例：

(2) fstat() - 通过文件描述符获取信息

  • 适用场景：已打开的文件（如通过 open() 获取的 fd）。

(3) lstat() - 不解析符号链接

  • 与 stat() 的区别：对符号链接文件返回链接本身的信息，而非目标文件。

宏含义
S_ISREG()普通文件
S_ISDIR()目录
S_ISCHR()字符设备
S_ISBLK()块设备
S_ISFIFO()命名管道（FIFO）
S_ISLNK()符号链接
S_ISSOCK()套接字

宏含义八进制值
S_IRUSR用户读权限0400

5. 注意事项

  1. 符号链接处理：
    • 用 lstat() 检查链接本身，stat() 会跟踪链接。
  2. 权限位兼容性：
    • 直接使用 0777 等八进制数可能降低可读性，建议用宏（如 S_IRUSR）。
  3. 时间戳精度：
    • st_atime/st_mtime 的精度取决于文件系统（可能为秒或纳秒）。
  4. 错误处理：
    • 检查函数返回值（如 stat() 返回 -1 时表示失败）。

6. 总结

<sys/stat.h> 的核心用途：

  1. 文件元数据获取：
    • stat(), fstat(), lstat() + struct stat。
  2. 文件类型判断：
    • S_ISDIR(), S_ISREG() 等宏。
  3. 权限管理：
    • 权限位宏（S_IRUSR）、chmod()。
  4. 文件系统操作：
    • mkdir(), mkfifo()。

适用于 文件管理工具、备份系统、权限监控 等场景。
