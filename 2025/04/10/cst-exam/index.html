✅ 为什么推荐在 svelte.config.js 中配置 alias？

SvelteKit 提供了官方支持的 alias 机制，它是专门为 Svelte 项目设计的，兼容性更好，适配更多开发工具链，比如：

  • ✅ 支持 $lib、$routes、$app 等特殊别名（这些只在 svelte.config.js 中能正常工作）

  • ✅ 能被

    Svelte 编译器

    正确识别，应用在：

    • import 路径中
    • Svelte 文件内的 src=""、href=""、action="" 等标签中

  • ✅ 自动传递给 Vite 和 TypeScript（通过 SvelteKit 内部桥接）

❗ 如果你只在 vite.config.js 中配置会怎样？

场景结果
JS/TS 文件中✅ 能识别 alias
.svelte 文件中 import ... from '@/...'❌ 有时不生效 或报错
<img src="@/assets/xxx.png" />❌ 会找不到资源
开发体验❌ VSCode 和类型提示经常报错（特别是路径跳转）

✅ 正确的做法

你应该把 alias 配在 svelte.config.js 的 kit.alias 中，SvelteKit 会自动传给 Vite 和 TypeScript。

👉 SvelteKit 项目推荐始终使用 svelte.config.js 来配置 alias，不建议只在 vite.config.js 中配置。

十二.上传文件

❗ 为啥「点击上传」必须要 input？

浏览器不会允许 JavaScript 随意访问本地文件系统 ——

点击上传只能靠 <input type="file"> 元素打开文件选择框，不管你隐藏得多深，底层都是它。

✅ 但你可以让 input 完全“隐身”！

  2. 在js逻辑中写dom操作,创建input标签,设置type=”file”,设置点击,加到body中去
